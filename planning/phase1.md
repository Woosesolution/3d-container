# Phase 1 - 可演示 MVP（含 3D 交互 + 自动装柜）

## 1. 目标

- 在不依赖后端的前提下，先交付完整可演示流程：
  - 输入柜体和产品参数
  - 生成托盘模板
  - 自动装柜
  - 手动拖拽调整
- 满足你要求：Phase 1 必须包含 3D 互动和自动装柜算法。

## 2. 范围

### 2.1 In Scope

- 参数输入：
  - 柜体（L/W/H、最大限重）
  - 规则（层数、分段、是否可放宽）
  - 产品（长宽高、密度或单重、件数、颜色）
- 算法能力：
  - 托盘候选生成（flat/mixed）
  - 自动装柜（基础启发式）
  - 三重检查（边界、碰撞、限重）
- 3D 互动：
  - Orbit 旋转/缩放
  - 拖拽吸附
  - 非法落位提示与回退
  - 点击模型查看参数摘要
- 结果展示：
  - 每托件数、托盘尺寸、托盘重量
  - 装载率、失败原因摘要

### 2.2 Out of Scope

- 账号登录、多租户、数据库持久化
- 审计日志和权限体系
- 复杂异形模型上传

## 4. 功能细化

### 4.1 Unique 模型定义（Phase 1）

- 每个产品模型通过以下维度实现唯一化：
  - 几何尺寸（L/W/H）
  - 颜色
  - 标签（产品名 + 规格）
- 当前采用参数化 Box 几何，不做异形几何。

### 4.2 自动装柜策略（Phase 1）

- 先排序：重量优先 + 体积优先 + 底面积优先。
- 放置策略：按层扫描可放置点。
- 每次试放执行：
  - `inside(container, item)` 越界检查
  - `intersects(a, b)` AABB 碰撞检查
  - `totalWeight <= maxWeight` 限重检查

### 4.3 手动交互策略（Phase 1）

- 拖拽开始记录原坐标。
- 拖拽中网格吸附，实时显示合法/非法状态色。
- 松手非法则自动回退到原位并提示原因。

## 5. 交付物

1. 可运行前端页面（本地静态服务）。
2. 托盘计算与装柜核心模块代码。
3. 3D 互动能力（拖拽、回退、提示）。
4. `testing/evidence/debug-history.md` 中至少 5 条关键场景记录。

## 6. 测试清单

1. 单产品基础场景：能自动生成托盘并正确装柜。
2. 多产品混合场景：无碰撞漏判。
3. 极限尺寸场景：超宽/超高必须拒绝。
4. 限重场景：超过总重时必须阻止放置。
5. 手动拖拽场景：非法释放必须回退。

## 7. 通过标准（Gate）

- 演示流程完整可跑通：
  - 输入 -> 生成 -> 自动装柜 -> 手动调整 -> 输出结果
- 无高优先级 bug：
  - 碰撞漏判
  - 越界漏判
  - 超重漏判
- 关键测试全部通过并记录证据。

## 8. 风险与对策

- 风险：Phase 1 范围大，容易延期。
  - 对策：先保正确性，再优化体验和性能。
- 风险：拖拽检测性能波动。
  - 对策：先做简单空间分桶，减少逐一碰撞检测次数。
