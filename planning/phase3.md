# Phase 3 - 算法与交互增强

## 1. 目标

- 在 Phase 1/2 稳定后提升“装得更好、解释更清晰、交互更顺畅”。
- 让结果不仅可用，还具备业务可解释性。

## 2. 范围

### 2.1 In Scope

- 自动装柜算法增强：
  - 更优候选排序
  - 层内 free-space 更新策略
  - 失败原因分类（尺寸/重量/冲突）
- 评分函数增强：
  - 空间利用率
  - 重量利用率
  - 订单覆盖率
  - 碎片惩罚
- 交互增强：
  - 模型锁定/解锁
  - 批量微调（可选）
  - 信息面板完善（来源参数 + 结果解释）
- 观测增强：
  - debug trace 结构化输出
  - 性能指标采样（放置次数、平均检测耗时）

### 2.2 Out of Scope

- 完整运筹优化求最优解（MIP/CP-SAT）
- 复杂物理仿真
- 异形模型自动摆放

## 3. 算法改进方向

### 3.1 评分函数

`score = a*spaceUtil + b*weightUtil + c*qtyCoverage - d*fragPenalty - e*violationPenalty`

要求：
- 权重配置化（便于调参）。
- 输出每个候选的评分明细，支持前端展示。

### 3.2 放置策略

- 在层内维护 `free-rectangles`。
- 新物体放置后更新可用矩形。
- 若失败，尝试旋转与次优候选点。

## 4. 交互改进方向

- 拖拽中显示“冲突对象 ID/名称”。
- 点击模型侧栏展示：
  - 来源产品参数
  - 托盘模板
  - 当前坐标与占用体积
- 锁定后不允许自动算法覆盖已锁定对象。

## 5. 交付物

1. 可配置评分模块。
2. 增强版自动装柜模块。
3. 增强版交互模块（锁定、解释面板）。
4. 性能与质量对比报告（相对 Phase 1）。

## 6. 测试清单

1. 同数据集下，Phase 3 装载率不低于 Phase 1。
2. 失败原因分类准确可复现。
3. 锁定对象不会被自动装柜覆盖。
4. 拖拽冲突提示对象信息正确。

## 7. 通过标准（Gate）

- 至少一个核心指标提升（装载率或失败率）。
- 所有新增交互测试通过。
- 无明显性能退化（平均交互帧率可接受）。

## 8. 风险与对策

- 风险：算法复杂化导致可维护性下降。
  - 对策：模块化 + trace + 回归基线。
- 风险：调参过度依赖单一场景。
  - 对策：固定测试集做 A/B 对比。
